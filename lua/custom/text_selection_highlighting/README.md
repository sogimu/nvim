# Text Selection Highlighting

Кастомный модуль Neovim для ручной подсветки выделенных участков текста разными цветами. Полезен при анализе больших файлов, когда нужно визуально группировать фрагменты кода или логи.

## Особенности

- **5 цветов подсветки**: зелёный, циан, маджента, оранжевый, тёмно-зелёный
- **Персистентность**: Подсветка привязана к буферу, не сбрасывается при открытии NvimTree или сплитов
- **Навигация**: Прыжки между созданными метками с центрированием экрана
- **Авто-запоминание цвета**: Повторное нажатие `Enter` использует последний выбранный цвет

## Функциональные клавиши

| Клавиша | Режим | Описание |
|---------|-------|----------|
| `<leader>h` | Visual | Выделить фрагмент и выбрать цвет (1-5) |
| `<leader>n` | Normal | Перейти к следующей метке |
| `<leader>N` | Normal | Перейти к предыдущей метке |
| `<leader>H` | Normal | Очистить все метки в текущем буфере |

## Установка

Модуль уже встроен в конфигурацию. Подключение происходит в `init.lua`:

```lua
require("custom.text_selection_highlighting")
```

## Использование

1. Выдели текст в визуальном режиме (`v` или `V`)
2. Нажми `<leader>h` (обычно `Пробел + h`)
3. Введи номер цвета (1-5) или нажми `Enter` для использования последнего цвета
4. Для навигации используй `<leader>n` и `<leader>N`
5. Для очистки всех меток в файле — `<leader>H`

## Технические детали

- Использует современный API `nvim_buf_set_extmark` для надежной привязки меток к буферу
- Хранит данные в памяти Lua-таблицы (не персистентно между сессиями)
- Не конфликтует с поиском и другими плагинами подсветки
